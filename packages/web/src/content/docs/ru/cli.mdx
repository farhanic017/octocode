CLI octocode по умолчанию запускает [TUI](/docs/tui) при запуске без каких-либо аргументов.

```bash
octocode
```

Но он также принимает команды, описанные на этой странице. Это позволяет вам программно взаимодействовать с octocode.

```bash
octocode run "Explain how closures work in JavaScript"
```

---

### tui

Запустите TUI octocode.

```bash
octocode [project]
```

#### Флаги

| Флаг                                     | Короткий | Описание                                                                      |
| ---------------------------------------- | -------- | ----------------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | Продолжить последний сеанс                                                    |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | Идентификатор сеанса для продолжения                                          |
| <nobr><code>{"--fork"}</code></nobr>     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |          | Промпт для использования                                                      |
| <nobr><code>{"--model"}</code></nobr>    | `-m`     | Модель для использования в виде поставщика/модели.                            |
| <nobr><code>{"--agent"}</code></nobr>    |          | Агент для использования                                                       |
| <nobr><code>{"--port"}</code></nobr>     |          | Порт для прослушивания                                                        |
| <nobr><code>{"--hostname"}</code></nobr> |          | Имя хоста для прослушивания                                                   |

## Команды

CLI octocode также имеет следующие команды.

---

### agent

Управляйте агентами для octocode.

```bash
octocode agent [command]
```

---

### attach

Подключите терминал к уже работающему внутреннему серверу octocode, запущенному с помощью команд `serve` или `web`.

```bash
octocode attach [url]
```

Это позволяет использовать TUI с удаленным сервером octocode. Например:

```bash
# Start the backend server for web/mobile access
octocode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
octocode attach http://10.20.30.40:4096
```

#### Флаги

| Флаг                                     | Короткий | Описание                                                                                         |
| ---------------------------------------- | -------- | ------------------------------------------------------------------------------------------------ |
| <nobr><code>{"--dir"}</code></nobr>      |          | Рабочий каталог для запуска TUI                                                                  |
| <nobr><code>{"--continue"}</code></nobr> | `-c`     | Продолжить последний сеанс                                                                       |
| <nobr><code>{"--session"}</code></nobr>  | `-s`     | Идентификатор сеанса для продолжения                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |          | Создать ответвление сеанса при продолжении (используйте с `--continue` или `--session`)          |
| <nobr><code>{"--password"}</code></nobr> | `-p`     | Пароль базовой аутентификации (по умолчанию `OCTOCODE_SERVER_PASSWORD`)                          |
| <nobr><code>{"--username"}</code></nobr> | `-u`     | Имя пользователя базовой аутентификации (по умолчанию `OCTOCODE_SERVER_USERNAME` или `octo`) |

### auth

Команда для управления учетными данными и входом в систему для провайдеров.

```bash
octocode auth [command]
```

---

#### login

octocode использует список провайдеров с [Models.dev](https://models.dev), поэтому вы можете использовать `octocode auth login` для настройки ключей API для любого поставщика, которого вы хотите использовать. Это хранится в `~/.local/share/octocode/auth.json`.

```bash
octocode auth login
```

Когда octocode запускается, он загружает поставщиков из файла учетных данных. И если в ваших средах определены какие-либо ключи или файл `.env` в вашем проекте.

---

#### list

Перечисляет всех проверенных поставщиков, которые хранятся в файле учетных данных.

```bash
octocode auth list
```

Или короткая версия.

```bash
octocode auth ls
```

---

#### logout

Выключает вас из провайдера, удаляя его из файла учетных данных.

```bash
octocode auth logout
```

#### add


#### auth

Аутентификация с помощью сервера MCP с поддержкой OAuth.

```bash
octocode mcp auth [name]
```

Если вы не укажете имя сервера, вам будет предложено выбрать один из доступных серверов с поддержкой OAuth.

Вы также можете перечислить серверы с поддержкой OAuth и их статус аутентификации.

```bash
octocode mcp auth list
```

Или используйте короткую версию.

```bash
octocode mcp auth ls
```

---

#### logout

Удалите учетные данные OAuth для сервера MCP.

```bash
octocode mcp logout [name]
```

---

#### debug

Отладка проблем с подключением OAuth для сервера MCP.

```bash
octocode mcp debug <name>
```

### models

Перечислите все доступные модели от настроенных поставщиков.

```bash
octocode models [provider]
```

Эта команда отображает все модели, доступные у настроенных вами поставщиков, в формате `provider/model`.

Это полезно для определения точного названия модели, которое будет использоваться в [вашем config](/docs/config/).

При желании вы можете передать идентификатор поставщика, чтобы фильтровать модели по этому поставщику.

```bash
octocode models anthropic
```

#### Флаги

| Флаг                                    | Описание                                                                          |
| --------------------------------------- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | Обновите кеш моделей на сайте models.dev.                                         |
| <nobr><code>{"--verbose"}</code></nobr> | Используйте более подробный вывод модели (включая метаданные, такие как затраты). |

Используйте флаг `--refresh` для обновления списка кэшированных моделей. Это полезно, когда к поставщику добавлены новые модели и вы хотите увидеть их в octocode.

```bash
octocode models --refresh
```

### run

Запустите octocode в неинтерактивном режиме, передав приглашение напрямую.

```bash
octocode run [message..]
```

Это полезно для создания сценариев, автоматизации или когда вам нужен быстрый ответ без запуска полного TUI. Например.

```bash "octocode run"
octocode run Explain the use of context in Go
```

Вы также можете подключиться к работающему экземпляру `octocode serve`, чтобы избежать холодной загрузки сервера MCP при каждом запуске:

```bash
# Start a headless server in one terminal
octocode serve

# In another terminal, run commands that attach to it
octocode run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

| <nobr><code>{"--session"}</code></nobr>  | `-s`     | Идентификатор сеанса для продолжения                                                             |
| <nobr><code>{"--fork"}</code></nobr>     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`)                    |
| <nobr><code>{"--share"}</code></nobr>    |          | Поделиться сеансом                                                                               |
| <nobr><code>{"--model"}</code></nobr>    | `-m`     | Модель для использования в виде поставщика/модели.                                               |
| <nobr><code>{"--agent"}</code></nobr>    |          | Агент для использования                                                                          |
| <nobr><code>{"--file"}</code></nobr>     | `-f`     | Файл(ы) для прикрепления к сообщению                                                             |

| <nobr><code>{"--port"}</code></nobr>     |          | Порт локального сервера (по умолчанию случайный порт)                                            |

### serve

Запустите автономный сервер octocode для доступа к API. Полный HTTP-интерфейс можно найти в [server docs](/docs/server).

```bash
octocode serve
```

При этом запускается HTTP-сервер, который обеспечивает доступ API к функциям octocode без интерфейса TUI. Установите `OCTOCODE_SERVER_PASSWORD`, чтобы включить базовую аутентификацию HTTP (имя пользователя по умолчанию — `octo`).

#### Флаги

| Флаг                                     | Описание                                                      |
| ---------------------------------------- | ------------------------------------------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | Порт для прослушивания                                        |
| <nobr><code>{"--hostname"}</code></nobr> | Имя хоста для прослушивания                                   |
| <nobr><code>{"--mdns"}</code></nobr>     | Включить обнаружение mDNS                                     |
| <nobr><code>{"--cors"}</code></nobr>     | Дополнительные источники браузера, позволяющие разрешить CORS |

### upgrade

Обновляет octocode до последней версии или определенной версии.

```bash
octocode upgrade [target]
```

Чтобы обновиться до последней версии.

```bash
octocode upgrade
```

Для обновления до определенной версии.

```bash
octocode upgrade v0.1.48
```

| Флаг                                   | Короткий | Описание                                                  |
| -------------------------------------- | -------- | --------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m`     | Используемый метод установки: local, npm, pnpm, bun, brew |

| `OCTOCODE_DISABLE_PRUNE`              | логическое значение | Отключить удаление старых данных                                                 |
| `OCTOCODE_DISABLE_TERMINAL_TITLE`     | логическое значение | Отключить автоматическое обновление заголовка терминала                          |

| `OCTOCODE_ENABLE_EXPERIMENTAL_MODELS` | логическое значение | Включить экспериментальные модели                                                |

### Экспериментальные функции

| `OCTOCODE_EXPERIMENTAL`                         | логическое значение | Включить экспериментальные функции под общим флагом    |
| `OCTOCODE_EXPERIMENTAL_ICON_DISCOVERY`          | логическое значение | Включить обнаружение значков                           |
| `OCTOCODE_EXPERIMENTAL_DISABLE_COPY_ON_SELECT`  | логическое значение | Отключить копирование при выборе в TUI                 |
| `OCTOCODE_EXPERIMENTAL_BASH_DEFAULT_TIMEOUT_MS` | число               | Таймаут по умолчанию для команд bash в мс              |
| `OCTOCODE_EXPERIMENTAL_OUTPUT_TOKEN_MAX`        | число               | Максимальное количество токенов вывода для ответов LLM |
| `OCTOCODE_EXPERIMENTAL_FILEWATCHER`             | логическое значение | Включить просмотр файлов для всего каталога            |
| `OCTOCODE_EXPERIMENTAL_OXFMT`                   | логическое значение | Включить форматтер oxfmt                               |
| `OCTOCODE_EXPERIMENTAL_LSP_TOOL`                | логическое значение | Включить экспериментальный инструмент LSP              |
| `OCTOCODE_EXPERIMENTAL_DISABLE_FILEWATCHER`     | логическое значение | Отключить просмотрщик файлов                           |
| `OCTOCODE_EXPERIMENTAL_EXA`                     | логическое значение | Включить экспериментальные функции Exa                 |
| `OCTOCODE_EXPERIMENTAL_LSP_TY`                  | логическое значение | Включить TY LSP для файлов python                      |
| `OCTOCODE_EXPERIMENTAL_PLAN_MODE`               | логическое значение | Включить режим плана                                   |
| `OCTOCODE_EXPERIMENTAL_BACKGROUND_SUBAGENTS`    | логическое значение | Включить фоновые задачи субагентов                     |
| `OCTOCODE_EXPERIMENTAL_EVENT_SYSTEM`            | логическое значение | Включить экспериментальную систему событий             |
| `OCTOCODE_EXPERIMENTAL_NATIVE_LLM`              | логическое значение | Включить нативный путь запросов LLM                    |
| `OCTOCODE_EXPERIMENTAL_PARALLEL`                | логическое значение | Включить параллельное выполнение веб-поиска            |
| `OCTOCODE_EXPERIMENTAL_SCOUT`                   | логическое значение | Включить субагент Scout                                |
| `OCTOCODE_EXPERIMENTAL_WORKSPACES`              | логическое значение | Включить поддержку рабочих областей                    |
